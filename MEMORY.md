## 核心方法论

### 知识迁移循环
1. **输入**：获取新知识（Python/英语等）
2. **拆解**：去语境化，提取逻辑骨架（BOM拆解）
3. **重构**：植入采购/供应链场景
4. **固化**：形成 SOP 或工具，变成可复用模块

### 三维进化分支
当一个模块"固化"后，触发以下路径之一：
- **分支一：纵向深潜** —— 在当前项目中持续挖掘，优化颗粒度与体验
- **分支二：横向迁移** —— 将已验证逻辑从旧载体迁移到新载体
- **分支三：抽象升维** —— 提取"元能力"，应用到更广阔领域

### 配置验证原则 ⚠️
**永远不要假设配置会按预期工作 —— 验证！**

**定时任务三步验证法：**
1. **手动测试脚本** - 确保脚本本身能正常执行
2. **检查配置正确性** - 确认 `isolated` + `agentTurn` + `now`
3. **查看执行历史** - 验证 duration 是否合理（>1s），检查实际效果

**警惕信号：**
- 执行时长 < 100ms → 可能只发消息，未真正执行
- 没有预期输出/文件变更 → 任务失败
- "感觉配置没问题" → 手动跑一遍再说

**避免"口头先生"：**
- 配置完立即测试
- 看到实际效果才算完成
- 文档记录验证方法

### 定时任务执行陷阱 ⚠️
**"口头先生"问题**：定时任务只发送消息，不执行命令

**错误配置（只说不做）：**
```json
{
  "sessionTarget": "main",
  "payload.kind": "systemEvent",
  "wakeMode": "next-heartbeat"
}
```
**结果：** 只在主会话显示消息，bash/脚本不会执行，duration 仅 7ms

**正确配置（真正执行）：**
```json
{
  "sessionTarget": "isolated",
  "payload.kind": "agentTurn",
  "wakeMode": "now"
}
```
**结果：** 启动独立子会话，完整执行命令，准时触发

**配置说明：**
- `sessionTarget="isolated"` - 启动独立子会话，有完整执行环境
- `payload.kind="agentTurn"` - AI 会真正执行命令（bash/python等）
- `wakeMode="now"` - 准时触发，不依赖心跳轮询

**案例教训：**
- ❌ 2026-02-07 天气任务失败：配置错误导致从未执行
- ❌ 2026-02-08 GitHub 备份失败：同样问题，脚本从未真正运行
- ✅ 2026-02-08 修复后：手动测试成功，commit 3aff6a7

**检查方法：**
- 查看执行时长：7ms = 只发消息，>1s = 真正执行
- 查看实际结果：脚本是否产生预期输出/文件变更
