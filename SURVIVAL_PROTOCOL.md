## 🛡️ 防护层级

### 第一层：预防（Reading Guard）

**强制规则：**
1. **读取任何文件前必须安检**
   ```bash
   bash /root/.openclaw/workspace/file-check.sh /path/to/file
   ```

2. **根据安检结果选择策略：**
   - **安全 (<50KB)** → 直接读取 ✅
   - **警告 (50-100KB)** → 确认后读取，或分段 ⚠️
   - **危险 (>100KB)** → 必须 offset/limit 或子会话 ⛔

3. **禁用行为：**
   - ❌ 不检查直接读取
   - ❌ 跳过安检
   - ❌ 抱"应该没事"的侥幸心理

---

### 第二层：监控（Session Health）

**健康指标：**
- 对话轮数（每次你来消息 = 1轮）
- 累计读取的文件大小
- 响应速度变化

**触发条件：**
- 对话超过 30 轮 → 开始关注
- 对话超过 50 轮 → **强烈建议重启**
- 读取了多个大文件 → **立即重启**

**提醒模板：**
```
💡 建议：当前对话已累计 N 轮，为避免 Context Overflow，
建议重启会话。你发送"重启"即可。
```

---

### 第三层：应急（Emergency Protocol）

**当出现以下情况时：**
1. 响应变慢
2. 思考时间过长
3. 收到奇怪的错误

**立即行动：**
1. 停止读取任何文件
2. 告诉你："🚨 检测到上下文压力过大，建议立即重启会话"
3. 不要试图"抢救"，直接重启

## 🔧 自主修复能力（v2.0 新增）

### ✅ 我可以自行处理的问题

#### 1. 网络故障
**问题：** 无法连接外网
**自动修复：**
- 重启网络服务（network/NetworkManager/systemd-networkd）
- 刷新 DNS（切换到 8.8.8.8/114.114.114.114）
- 测试连通性
- 如果仍失败，记录日志并建议你检查

#### 2. 硬件问题
**问题：** 系统异常、性能下降
**自动修复：**
- 检查内核错误日志
- 查看硬件温度（如果可用）
- 检查磁盘 I/O
- 查看系统负载
- 尝试：
  - 清理资源（缓存、临时文件）
  - 重启异常服务
  - 记录详细信息供你分析

#### 3. 服务配置错误
**问题：** 服务启动失败、运行异常
**自动修复：**
- 查看服务日志
- 尝试重启服务
- 如果是可选服务（如 ipmi、kdump、mcelog）：
  - 直接禁用并停止
- 如果是核心服务：
  - 查看错误日志
  - 尝试修复配置
  - 如果无法修复，记录详细信息并通知你

#### 4. 内存不足
**自动修复：**
- 清理系统缓存：`echo 3 > /proc/sys/vm/drop_caches`
- 清理应用缓存（npm, apt 等）
- 如果仍不足，通知你

#### 5. 磁盘空间不足
**自动修复：**
- 清理临时文件（/tmp, /var/tmp）
- 清理日志文件（压缩大日志、删除旧日志）
- 清理包管理器缓存（apt, npm）
- 清理 OpenClaw 临时文件

#### 6. 进程异常
**自动修复：**
- 清理僵尸进程
- 重启失败的服务
- 如果 OpenClaw 挂了，自动重启

#### 7. 日志堆积
**自动修复：**
- 压缩大日志文件（> 10MB）
- 删除旧日志（> 30天）
- 清理 OpenClaw 日志（> 50MB）
- 清理系统日志（journalctl vacuum）

## 📋 分级巡检系统（v2.0）

### 每天例行检查（04:00）
- **时长**：< 30秒
- **检查**：内存、磁盘、进程、网络、错误日志
- **自动修复**：
  - 内存 > 85% → 清理缓存
  - 磁盘 > 85% → 清理临时文件
  - OpenClaw 挂了 → 重启
- **报告**：仅当发现问题时发送

### 每周中度检查（周日 04:00）
- **时长**：< 2分钟
- **检查**：系统详情、进程、服务、日志、网络、工作区
- **自动修复**：
  - 清理僵尸进程
  - 重启失败的服务
  - 清理/压缩日志文件
  - 清理临时文件
  - 尝试恢复网络
  - 清理 OpenClaw 日志
- **报告**：无论成功失败都发送（含修复详情）

### 每月大度检查（1日 04:00）
- **时长**：5-10分钟
- **检查**：硬件、存储、网络、资源、安全、趋势
- **自动修复**：
  - 深度清理（包管理器、日志、临时文件）
  - 禁用不需要的服务
  - 清理各种缓存
- **报告**：完整审计报告 + 趋势分析
