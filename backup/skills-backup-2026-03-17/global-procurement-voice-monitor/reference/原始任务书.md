# 全球采购真实心声监控任务书 v2.0

## Global Procurement Voice Monitor — Task Brief

---

## 一、任务概述

### 1.1 任务目标

面向全球采购从业者群体，系统性收集其在公开社交平台与职业社区上的真实心声（抱怨、吐槽、求助、宣泄、反思等），通过无预设归纳提炼出高频共性问题，最终为采购协会的靶向培训计划提供数据支撑。

### 1.2 核心原则

**先收集，后归纳，再分类。**

本任务严禁"定向证据采集"——即先预设结论再找支撑素材。正确流程为：

1. 广泛采集原始语料（不带预判地大量收集）
2. 对语料做自然聚类（让主题从数据中涌现）
3. 将涌现出的主题归入分析框架（用框架定位，不用框架预设）
4. 统计各维度/指标的声量分布与情绪浓度
5. 基于数据输出洞察，对接培训设计

### 1.3 适用对象

本任务书交付给执行团队（数据采集员、内容分析师、培训设计师）作为操作手册使用。

## 二、分析框架：6维度 × 3指标坐标系

> **使用说明：** 以下框架是"分类坐标系"，不是"预设结论"。采集到的每一条真实心声，按照以下6个维度进行标注和归类。每个维度下的3个指标是初始分类锚点，实际执行中如果数据涌现出框架未覆盖的新主题，应及时新增指标并反馈。

### 框架总览

| 维度编号 | 维度名称 | 核心问题 | 指标1 | 指标2 | 指标3 |
|----------|----------|----------|-------|-------|-------|
| D1 | 痛点主题 | 他们在痛什么？ | 工作负荷与效率 | 薪酬与职业发展 | 专业能力挑战 |
| D2 | 抱怨对象 | 火对谁发？ | 供应商端 | 内部上级/跨部门 | 制度与系统 |
| D3 | 情绪光谱 | 情绪浓度如何？ | 吐槽调侃型（黄灯） | 焦虑疲惫型（橙灯） | 崩溃绝望型（红灯） |
| D4 | 事态阶段 | 问题走到哪了？ | 困局僵持中 | 妥协推进中 | 已解决/复盘中 |
| D5 | 地域文化 | 在哪里发声？ | 中国 | 印度 | 北美/欧洲 |
| D6 | 角色画像 | 谁在发声？ | 执行层（0-3年） | 中层管理（3-8年） | 高层决策（10年+） |

### 各维度标注说明

**D1｜痛点主题** — 这条心声的核心痛点属于什么类别？

| 指标 | 判定标准 | 典型信号词（仅供参考，不限于此） |
|------|----------|-------------------------------|
| D1.1 工作负荷与效率 | 关于工作量、加班、流程繁琐、人手不够的内容 | 待采集后补充 |
| D1.2 薪酬与职业发展 | 关于薪资、晋升、职业路径、价值感的内容 | 待采集后补充 |
| D1.3 专业能力挑战 | 关于技能不足、数字化转型、新要求适应难的内容 | 待采集后补充 |

**D2｜抱怨对象** — 这条心声的不满指向谁/什么？

| 指标 | 判定标准 | 典型信号词（仅供参考，不限于此） |
|------|----------|-------------------------------|
| D2.1 供应商端 | 抱怨供应商交期、质量、态度、配合度等 | 待采集后补充 |
| D2.2 内部上级/跨部门 | 抱怨领导、PMC、研发、销售、办公室政治等 | 待采集后补充 |
| D2.3 制度与系统 | 抱怨审批流程、ERP/SRM系统、合规要求等 | 待采集后补充 |

**D3｜情绪光谱** — 这条心声的情绪浓度是什么级别？

| 指标 | 判定标准 | 判定线索 |
|------|----------|----------|
| D3.1 吐槽调侃型（🟡） | 用幽默、自嘲、段子方式表达，尚可承受 | 语气轻松、带表情包、评论区以"哈哈"共鸣为主 |
| D3.2 焦虑疲惫型（🟠） | 直接表达疲惫、压力、力不从心，有倾诉感 | 内容较长、语气沉重、评论区以"抱抱""我也是"为主 |
| D3.3 崩溃绝望型（🔴） | 出现离职意向、转行念头、对行业的否定 | 明确提及离开、后悔入行、身心健康受损 |

**D4｜事态阶段** — 这条心声描述的问题处于什么状态？

| 指标 | 判定标准 | 判定线索 |
|------|----------|----------|
| D4.1 困局僵持中 | 正在经历，看不到出路 | "怎么办""求助""stuck" |
| D4.2 妥协推进中 | 勉强推动，但不满意 | "只能这样了""先凑合""no choice" |
| D4.3 已解决/复盘中 | 问题关闭，正在反思 | "分享经验""lessons learned""踩坑记录" |

**D5｜地域文化** — 发声者所在地区？

| 指标 | 判定方法 |
|------|----------|
| D5.1 中国 | 中文平台（知乎/小红书/脉脉）发帖；或内容中明确提及中国工作场景 |
| D5.2 印度 | LinkedIn印度采购社群；或内容中明确提及印度工作场景 |
| D5.3 北美/欧洲 | 英文平台发帖；或内容中明确提及北美/欧洲工作场景 |

> 注：如发声者地域无法判定，标注为"未知"。如涌现出东南亚、中东等高频地区，应新增子指标。

**D6｜角色画像** — 发声者属于哪个层级？

| 指标 | 判定方法 |
|------|----------|
| D6.1 执行层（0-3年） | 自述为采购员/buyer/assistant；或内容反映执行性工作场景 |
| D6.2 中层管理（3-8年） | 自述为采购经理/category manager；或内容涉及团队管理+供应商策略 |
| D6.3 高层决策（10年+） | 自述为采购总监/VP/CPO；或内容涉及战略规划+组织变革 |

> 注：如角色层级无法判定，标注为"未知"。

## 四、分类与标注方法论

### 4.1 标注流程

```
原始语料 → 通读理解 → 六维度标注 → 交叉校验 → 入库
```

**Step 1：通读理解**
- 完整阅读原帖及核心评论
- 不带预判地理解发声者在表达什么

**Step 2：六维度标注**
- 对每条语料，在D1-D6六个维度上分别打标签
- 一条语料可以在同一维度下命中多个指标（如同时涉及D2.1供应商和D2.2跨部门）
- 如果某条语料在某个维度上无法判定，标注"不适用"或"未知"

**Step 3：交叉校验**
- 每条语料由至少2名标注人独立标注
- 标注不一致的条目进行讨论达成共识
- 标注一致率低于80%时，需回溯检查标注标准是否清晰

**Step 4：入库**
- 标注完成的语料录入数据库/表格，进入后续分析环节

### 4.2 新主题涌现处理规则

在标注过程中，如果反复遇到无法归入现有3个指标的内容，说明有新主题正在涌现。处理规则如下：

1. 单条语料无法归类 → 暂时标注为"其他"并备注关键词
2. "其他"类在某个维度下累计超过总量的10% → 触发新指标审议
3. 审议确认后 → 新增指标（D1.4、D2.4……），框架从"M×3"动态扩展为"M×N"
4. 新增指标需回溯标注此前的"其他"类语料

> **这一机制是保证"数据驱动"而非"框架预设"的关键。**

### 4.3 采集量目标

| 周期 | 每平台最低采集量 | 总量目标 | 说明 |
|------|-----------------|----------|------|
| 每周 | 30条/平台 | 200-300条/周 | 覆盖7-9个平台 |
| 每月汇总 | — | 800-1200条/月 | 去重后有效语料 |
| 每季度报告 | — | 2500-3500条/季度 | 完整分析基础 |

> 以上为最低标准。高频事件期间（如重大政策变化）可临时加大采集量。

## 六、从心声到培训的转化机制

### 6.1 转化路径

```
季度心声数据
    ↓
识别TOP痛点（按声量×情绪浓度加权排序）
    ↓
按角色层级拆分（同一痛点，不同层级需要不同培训）
    ↓
按地域文化适配（同一培训主题，不同地区需要不同案例和交付方式）
    ↓
输出培训计划（主题 × 群体 × 地域 × 形式）
    ↓
执行培训
    ↓
下一周期继续监控心声，评估培训效果是否体现在声量/情绪变化上
```

### 6.2 培训计划输出模板

> **以下表格在每个季度末由分析结果填充，此处展示结构。**

| 序号 | 培训主题 | 数据来源（对应心声维度/指标） | 目标群体（角色层级） | 适用地域 | 建议形式 | 优先级 |
|------|----------|---------------------------|---------------------|----------|----------|--------|
| 1 | （由数据决定） | D_._ | 执行层/中层/高层/全层级 | 中国/印度/北美/全球 | 工作坊/在线课/案例研讨/辅导 | P0/P1/P2 |
| 2 | | | | | | |
| 3 | | | | | | |
| ... | | | | | | |

### 6.3 效果追踪闭环

培训执行后，在下一监控周期中重点观察：

- 对应痛点的声量是否下降
- 对应情绪光谱是否从🔴🟠向🟡迁移
- 是否出现"培训有用"或"学到了"的正面反馈
- 是否有新问题因培训介入而被激活（如：培训让采购人意识到更多问题，短期内声量反而上升）

| 角色 | 职责 | 人数建议 |

## 八、质量控制清单

- [ ] 采集量是否达到本月最低标准
- [ ] 各平台覆盖是否均衡（无严重偏科）
- [ ] 三个地域（中国/印度/北美欧洲）是否都有足够语料
- [ ] 标注交叉校验一致率是否≥80%
- [ ] "其他"类占比是否超过10%阈值（如超过需审议新增指标）
- [ ] 月度简报所有结论是否都有原始语料支撑（可回溯）
- [ ] 是否存在"先有结论后找证据"的反向操作（严禁）
- [ ] 培训需求信号是否直接从数据中推导（而非主观臆断）
