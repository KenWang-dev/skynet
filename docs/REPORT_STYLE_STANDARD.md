## 🎨 排版规范详解

### 1. 报告头部

```
📊 **[报告名称]**
**YYYY-MM-DD | 过去 [时间窗口]**
```

**要素**：
- 📊 图标 + **报告名称**（加粗）
- **日期 | 时间窗口**（加粗）
- 时间窗口示例：过去 24 小时、过去 7 天、2026年3月等

### 2. 大板块标题

```
**【板块名称】**
```

**要点**：
- 使用【】包裹
- 不加粗
- 每个大板块代表一个主要分类（如：中国部分、美国部分、技术视角、资本视角）

### 3. 子板块标题

```
**子板块名称**
```

**要点**：
- **加粗**
- 代表子分类（如：政策动态、资金支持、产业园区）
- 简洁明确

### 4. 新闻/信息条目（核心）

每个新闻/信息是**独立的语义块**，格式如下：

```
**- 新闻/信息标题**

　　**简要解析**：详细解释内容，说明重要性或影响。

　　📎 **来源**：[来源名称](链接地址)
```

**要点**：
- ✅ **- 新闻标题**（加粗）
- ✅ 　　**简要解析**（缩进 + 加粗）
- ✅ 　　📎 **来源**（缩进 + 加粗）
- ✅ 每个新闻之间有空行

**缩进**：
- 使用两个全角空格 `　　`
- 突出层级关系

### 5. 空行规则（⚠️ 重要！）

**⚠️ 教训总结**：在这个问题上栽了很多跟头！必须严格遵守！

**✅ 正确做法**：使用**真正的空行**（两个换行符）

```markdown
**- 新闻1**
　　**简要解析**...
　　📎 **来源**...


**- 新闻2**
　　**简要解析**...
　　📎 **来源**...


**子板块2**
```

**❌ 错误做法**：使用全角空格单独一行

```markdown
**- 新闻1**
　　**简要解析**...
　　📎 **来源**...

　
　
　

**- 新闻2**
```

**技术说明**：
- ✅ **正确**：在两个内容之间放两个换行符 `\n\n`
- ❌ **错误**：使用全角空格 `　` 单独一行（会被 message 工具压缩）
- ❌ **错误**：连续多个换行符 `\n\n\n`（会被压缩为 `\n\n`）

**在 message 工具中**：
- 直接按两次回车，产生一个空行
- 不要用全角空格
- 不要用连续换行

**规则**：
- ✅ 不同新闻之间：空行（两个换行符）
- ✅ 不同子板块之间：空行（两个换行符）
- ✅ 不同大板块之间：分隔符 + 空行（两个换行符）

### 6. 分隔符

```
　

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　
```

**要点**：
- 横线上方：空行
- 横线下方：空行
- 用于分隔主要板块（如中美部分）

### 7. 总结部分

```
**核心洞察**

基于以上信息的总结分析，提出关键洞察和建议。

　

**一句话总结**

简要总结，一句话概括核心要点。
```

### 8. 报告结尾

```
**报告完成！**

明天 [时间] 见！🪭
```

**要点**：
- **报告完成！**（加粗）
- 明天 [时间] 见！🪭
- 时间要匹配实际执行时间

## 🎯 核心原则

### 1. 层次清晰
- 大板块 → 子板块 → 新闻条目
- 每一层有明确的视觉区分

### 2. 呼吸感
- 语义块之间有空行
- 避免信息过于密集

### 3. 突出重点
- 标题加粗
- 简要解析加粗
- 来源加粗
- 缩进突出层级

### 4. 信息完整
- 每条新闻都有标题、解析、来源
- 来源链接可点击查看详细内容

### 5. 易读性
- 结构一致
- 格式统一
- 视觉舒适

## 🔧 实施要点

### 生成报告时必须检查：

- [ ] 报告标题正确（📊 **报告名称**）

- [ ] 大板块标题（**【板块名称】**）
- [ ] 子板块标题（**子板块名称**，加粗）
- [ ] 新闻标题加粗（**- 新闻标题**）

- [ ] 来源缩进+加粗（`　　📎 **来源**`）
- [ ] 每个新闻之间有空行
- [ ] 子板块之间有空行
- [ ] 分隔符横线上下有空行
- [ ] 每条新闻都有来源链接
- [ ] 结尾时间正确
