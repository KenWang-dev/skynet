# 监控任务严格输出要求

## 🎯 核心原则

**所有监控任务必须：**
1. ✅ 直接输出报告内容
2. ❌ 不要有任何开场白
3. ❌ 不要有思考过程描述
4. ❌ 不要有准备工作说明
5. ❌ 不要说"我来执行"、"让我搜索"等

---

## ✅ 正确示例

```
📊 供应链风险日报 - 2026-03-10

🔴 核心发现：霍尔木兹海峡危机爆发

**红灯事件（4项）**
...
```

---

## ❌ 错误示例

```
我来执行供应链风险日报任务。
首先让我读取相关技能文档了解框架。
现在开始搜索过去24小时的风险指标...

📊 供应链风险日报
...
```

---

## 📋 所有任务的"严格输出要求"模板

复制以下内容到每个监控任务的 payload 末尾：

```markdown
**⚠️ 严格输出要求**：
- 直接输出报告内容，不要有任何开场白、思考过程、准备工作的描述
- 不要说"我来执行"、"让我搜索"、"正在分析"之类的话
- 第一句话就直接开始报告内容
- 只输出最终的报告，不要有过程描述
```

---

## 🔧 应用到任务

使用以下命令更新任务：

```bash
# 读取当前任务配置
cron list | grep "任务名称"

# 更新任务，添加"严格输出要求"
# (使用 cron update 命令)
```
