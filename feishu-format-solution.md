## 问题描述

飞书 message 工具发送的消息缺少空行，所有内容挤在一起，严重影响可读性。

## 根本原因

**飞书 message 工具的后端限制**：
- **短消息（< 1000 字）**：格式正常保留
- **长消息（> 2000 字）**：后端会重新排版，**压缩连续空行**，导致格式被破坏

## 解决方案

### 1. 空行格式

**正确格式**：
```markdown
内容A

　

内容B（有空行效果）
```

**要点**：
- 换行
- **单独一行**放全角空格（　）
- 再换行

### 2. 拆分发送

**周报拆分方案**（4 条独立消息）：
- **消息 1**：标题 + 核心复盘 + 红灯事件
- **消息 2**：重点品类趋势
- **消息 3**：上游原材料 + 宏观环境
- **消息 4**：战略决策建议

**每条消息**：
- 控制在 1000 字以内
- 使用正确的空行格式
- 保持相对完整的内容结构

## 经验教训

1. **不要假设工具会按预期工作** —— 飞书 message 工具对长消息有特殊处理
2. **格式验证很重要** —— 短消息正常不代表长消息也正常
3. **拆分是唯一可靠方案** —— 对于超长内容，必须拆分成多条消息
