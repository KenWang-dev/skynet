## 5. 处理

### 监控维度（待用户细化）

#### 维度 1：供需关系 (Supply & Demand)
- 产能变化（扩产/减产/停产）
- 需求波动（爆款产品/新兴应用）
- 库存水位（去库存/补库存）

#### 维度 2：价格趋势 (Price Trends)
- 原材料价格（硅晶圆、铜、铝）
- 元器件价格（芯片、被动元件）
- 汇率影响（美元/人民币/新台币）

#### 维度 3：技术突破 (Technology)
- 新材料/新工艺（3nm、2nm、先进封装）
- 替代方案（SiC vs GaN、不同技术路线）
- 标准变更（接口标准、通信协议）

#### 维度 4：地缘政治 (Geopolitics)
- 贸易政策（出口管制、关税调整）
- 制裁/禁运（实体清单、技术封锁）
- 战略竞争（中美科技战、芯片法案）

#### 维度 5：企业动态 (Corporate)
- 重大并购/重组
- 产能投资/建厂计划
- 供应链调整（迁厂/多元化）

#### 维度 6：外部生态 (External Ecosystem)
- 顶级分析（行业领袖、专家研判）
- 投资机构（VC/PE 趋势报告）
- 社区反馈（工程师、采购员口碑）

---

### 情报处理逻辑

1. **价值过滤**：剔除噪音，保留高信噪比信息
2. **事实校验**：交叉验证多个来源
3. **趋势对比**：与历史数据对比，识别异常
4. **影响评估**：对采购决策的影响程度（高/中/低）

---

## 6. 输出

### 报告结构

#### 顶部：Top 5 核心情报
- 提炼最具影响力的 5 条动态
- 每条包含：**事件 + 时间 + 影响评估**
- 按重要性降序排列

#### 中部：维度结构化详述
- 按 6 大维度分组
- 每个维度：**事实 + 分析 + 趋势**
- 简洁明了，便于扫读

#### 底部：行动建议（可选）
- 基于情报的采购建议
- 风险预警（高/中/低）
- 数据溯源（原始链接）

### 输出格式要求

- **语言**：简体中文
- **术语**：核心缩写保留英文（如 SiC、GaN、IC），首现加注中文
- **排版**：
  - 关键信息**加粗**
  - 段落间空行分隔
  - 多级缩进显示层级
  - 使用符号标注趋势（↑上升、↓下降、→持平、⚠️风险）
- **长度控制**：宁缺毋滥，无重大动态时明确说明

## 7. 约束

### 边界条件

**时效性**：
- 每日简报：早上 8:00 前发送
- 紧急情报：实时推送（Tavily 搜索结果中高影响事件）

**资源限制**：
- API 额度：Tavily 每日限额
- 计算成本：单次搜索 < 10 秒
- Token 消耗：报告长度 < 2000 tokens

**质量要求**：
- 准确率：核心事实准确率 > 95%
- 覆盖面：6 大维度至少覆盖 4 个
- 时效性：新闻不超过 24 小时

**合规性**：
- 数据来源合法（公开渠道）
- 不涉及内幕信息
- 遵守数据使用条款

## 8. 异常

### 失败预案

**数据源失败**：
- Tavily API 挂了 → 降级到 Brave Search
- 搜索无结果 → 扩大时间窗口（48 小时）
- 关键源不可用 → 使用备用源

**分析失败**：
- 搜索结果噪音太大 → 提高过滤阈值
- 分类失败 → 手动归类 + 记录失败案例
- 趋势判断不确定 → 标注「待验证」+ 持续追踪

**发送失败**：
- 飞书 API 超时 → 重试 3 次
- 消息过长 → 分批发送
- 接收方不可达 → 备用渠道（邮件/Telegram）

**超时处理**：
- 单次搜索超时 > 30 秒 → 跳过该源
- 整体流程超时 > 5 分钟 → 发送部分结果 + 标注「未完成」
