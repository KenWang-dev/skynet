# 飞书格式检查 Skill

## ⚠️ 飞书的两个软肋

### 1. 加粗 ≠ Markdown

**问题**：飞书普通文本不支持 `**加粗**` 语法

**解决方案**：必须用**卡片模式**

**触发方法**：消息末尾添加代码块
```
\n```\n🪭\n```
```

### 2. 空行用全角空格

**问题**：连续空行会被飞书压缩

**解决方案**：使用全角空格 `　` 作为段落分隔

---

## 📋 格式检查清单

在发送报告前，检查以下项目：

### ✅ 必须项

| 检查项 | 要求 | 示例 |
|--------|------|------|
| 卡片触发器 | 消息末尾有 `\n```\n🪭\n``` | 见下方示例 |
| 段落分隔 | 使用全角空格 `　` | `段落1　段落2` |
| 标题格式 | 使用 `##` 或 `###` | `## 【板块名称】` |
| 加粗格式 | 使用 `**文字**` | `**重要信息**` |

### ✅ 推荐项

| 检查项 | 要求 | 说明 |
|--------|------|------|
| 分隔线 | 使用 `---` | 区分大板块 |
| 列表缩进 | 使用 `　` 缩进 | 层级清晰 |
| 信息密度 | 每段不超过 3 行 | 避免信息过密 |

## 常见错误

| 错误 | 表现 | 修复 |
|------|------|------|
| 加粗不生效 | `**文字**` 显示原样 | 添加卡片触发器 |
| 段落粘连 | 信息挤在一起 | 用全角空格分隔 |
| 信息过密 | 看起来像一堵墙 | 增加分隔线和空格 |

---

**记住**：飞书格式 = 卡片模式 + 全角空格 + 合理分层 📝
