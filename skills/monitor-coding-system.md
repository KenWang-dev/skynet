## 📊 编码原则 v2.0

### 1️⃣ 字母分配规则（重构）

**AI类**：希腊字母系统
- **Alpha（阿尔法）**: Karpathy AI博客精选
- **Beta（贝塔）**: AI三巨头监控
- **Gamma（伽马）**: （预留扩展）

**采购类**：普通字母，从 A 开始
- **A**: 电子供应链
- **B**: 供应链风险
- **C**: 政策与法规
- **D**: 宏观财务
- **E**: 行业市场
- **F**: AI采购最佳实践
- **G**: 采购心声
- **H**: ESG绿色采购
- **I**: 供应商生态系统

---

### 2️⃣ 数字规则（保持不变）

**同一个监控任务，不同频率使用同一字母**：
- **1**: 日报（Daily）
- **2**: 周报（Weekly）
- **3**: 月报（Monthly）

**示例**：
- **C1**: 政策与法规监控日报
- **C2**: 政策与法规监控周报
- **C3**: 政策与法规监控月报

---

### 3️⃣ 时间分配规则（保持不变）

**早晨时段（7:00-7:30）**：6个日报
- 每5分钟一个任务
- 7:00, 7:05, 7:10, 7:15, 7:20, 7:25

**白天时段（7:45-8:35）**：周报
- 每5-10分钟一个任务
- 避免时间冲突

## 📝 编码规则总结 v2.0

1. **AI 类使用希腊字母**：Alpha, Beta, Gamma...（避免与采购类冲突）
2. **采购类使用普通字母**：A, B, C, D, E, F, G, H, I...
3. **数字代表频率**：1=日报，2=周报，3=月报
4. **同一任务同一字母**：不同频率用同一字母+不同数字
5. **时间顺序分配**：7:00开始，每5分钟一个，避免冲突

## 🚀 扩展规则 v2.0

### 新增任务编码规则

**新增AI类任务**：
- 使用希腊字母：Delta, Epsilon, Zeta...
- 示例：新增AI技术前沿监控 → Gamma1（日报）

**新增采购类任务**：
- 使用普通字母：J, K, L...（按顺序）
- 确定任务本质（活下去/活得好/活得久）
- 如有多频率，使用同一字母+不同数字

**示例**：
- 新增采购创新案例 → J2（周报）
- 新增供应商财务风险 → K1（日报）
